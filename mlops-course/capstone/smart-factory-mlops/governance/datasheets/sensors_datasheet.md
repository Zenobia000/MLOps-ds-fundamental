# Datasheet：設備感測器時序資料集（Sensors）

> 格式參考 *Datasheets for Datasets*（Gebru et al., 2021）。本說明書描述用於預測性維護
> 的感測器時序資料集，教學以 `datasets/toy_sensors.csv` 示例，真實情境對應 NASA C-MAPSS。

## 1. 動機（Motivation）

- **建立目的**：訓練與評估設備故障預測模型，支撐 Smart Factory 預測性維護。
- **建立者 / 出資者**：MLOps 平台組（教學用途）。
- **資助來源**：內部教學專案，無外部資助。

## 2. 組成（Composition）

| 項目 | 內容 |
| :--- | :--- |
| 每筆代表 | 單一機台於某時間點的感測器讀數快照 |
| 實體鍵 | `machine_id`（如 `machine_01`） |
| 時間戳 | `event_timestamp`（每小時一筆） |
| 特徵 | `temperature`（°C）、`vibration`（mm/s）、`current`（A） |
| 標籤 | `failure`（0 = 正常，1 = 故障） |
| 規模（玩具） | 多台機器 × 數百時間點；故障為少數類別（不平衡） |
| 缺失值 | 玩具資料無缺失；真實資料可能因感測器離線而缺漏 |
| 敏感資料 | 無個資，無人員相關資料 |

## 3. 蒐集流程（Collection Process）

- **取得方式**：由設備 IoT 感測器經邊緣閘道採集，定時上傳至時序資料庫。
- **取樣頻率**：每小時聚合一次（真實系統可為秒級，入庫前降採樣）。
- **時間範圍**：玩具資料自 2024-01-01 起連續數日。
- **品質控管**：入庫前做範圍檢查（溫度/振動/電流的物理合理區間）與去重。

## 4. 前處理 / 清洗 / 標註（Preprocessing）

- 清洗：移除超出物理量程的異常讀數、補齊或標記缺失時間點。
- 特徵工程：滾動均值、滾動標準差等於 Feast feature view 產生（見 `feature_repo/`）。
- 標註：`failure` 由維護工單回填（故障發生時間對齊感測讀數），須注意標註時間對齊以避免洩漏。

## 5. 使用建議（Uses）

- **適用**：故障預測、異常偵測、剩餘壽命（RUL）估計的特徵來源。
- **不適用**：跨機種直接遷移（不同設備物理特性不同）；不可用於人員監控。
- **已知偏誤**：
  - 各機台資料量不均，少數機台主導分布。
  - 故障樣本稀少，需以重採樣 / 加權處理。
  - 感測器隨時間老化（drift），歷史與近期分布可能不一致。

## 6. 分布與維護（Distribution & Maintenance）

- **散布範圍**：內部教學用，玩具資料隨 repo 提供；真實資料（C-MAPSS）依其授權自行取得，放 `data/external/`。
- **版本控管**：以 DVC 追蹤資料版本，Git 僅存指標檔。
- **維護者**：MLOps 平台組。
- **更新頻率**：線上資料持續累積；reference 快照隨模型版本刷新（見 `monitoring/evidently/README.md`）。
- **聯絡方式**：data-governance@smartfactory.example（教學示例）。

## 7. 法遵與隱私（Compliance）

- 本資料集不含個人資料，GDPR / 個資法不直接適用。
- 設備與製程資料可能屬營業秘密，存取需依內部權限控管，不得外流。
