# EU AI Act 風險評估：Smart Factory MLOps

> 依歐盟《人工智慧法》（Regulation (EU) 2024/1689, "AI Act"）的風險分級框架，
> 針對本 capstone 的三個 AI 系統進行風險評估與義務對應。
> **聲明**：本文件為教學範例，非法律意見；實際合規須由法務與合規團隊評估。

## 1. AI Act 四級風險框架

| 風險等級 | 定義 | 主要後果 |
| :--- | :--- | :--- |
| **不可接受風險**（Unacceptable） | 社會評分、潛意識操控、即時遠端生物辨識等 | **禁止** |
| **高風險**（High-risk） | Annex III 清單（關鍵基礎設施、教育、就業、執法…）或作為受規範產品的安全元件 | 嚴格義務（風險管理、資料治理、技術文件、人工監督、登錄等） |
| **有限風險**（Limited） | 與人互動 / 生成內容的系統（聊天機器人、deepfake） | 透明度義務（告知使用者正在與 AI 互動） |
| **最小風險**（Minimal） | 其餘大多數 AI（垃圾郵件過濾、推薦等） | 無強制義務，鼓勵自願行為準則 |

## 2. 本專案 AI 系統清單與分級

| 系統 | 用途 | 分級 | 理由 |
| :--- | :--- | :--- | :--- |
| 預測性維護（tabular/XGBoost） | 預測設備故障機率輔助維護排程 | **最小 / 有限風險** | 對象為設備非人員、非 Annex III 高風險場景、非安全元件的合格性認證；不影響個人權利。 |
| 需求預測（timeseries） | 預測產能需求供生產規劃 | **最小風險** | 純營運規劃輔助，無個人資料、無高風險場景。 |
| 視覺瑕疵檢測（vision/ResNet） | 產線零件瑕疵初篩 | **有限風險**（條件性） | 一般品檢輔助屬有限風險；**若**用於受規範產品（如機械、醫材）安全元件的合格性判定，則可能落入**高風險**，須重新評估。 |

> **關鍵判斷**：三系統皆**不涉及個人資料、不做人員評估、非生物辨識、非 Annex III 清單**，
> 故預設不屬高風險。視覺檢測若延伸至安全關鍵零件認證，須升級評估。

## 3. 適用義務對應

### 3.1 有限風險義務（適用本專案）

| 義務 | 對應落實 |
| :--- | :--- |
| 透明度（告知 AI 參與決策） | Model Card 載明用途與限制；UI/報表標示「AI 輔助判斷，須人工複核」。 |
| 紀錄留存 | MLflow 紀錄訓練 run、參數、指標；DVC 追蹤資料版本。 |
| 人工監督 | 維護決策與品檢放行保留 human-in-the-loop，AI 僅作建議。 |

### 3.2 若升級為高風險須補齊（前瞻準備）

| Art. | 義務 | 本專案準備狀態 |
| :--- | :--- | :--- |
| Art. 9 | 風險管理系統 | 本文件 + 監控閉環（部分） |
| Art. 10 | 資料治理（代表性、無偏） | Datasheet（`governance/datasheets/`） |
| Art. 11/技術文件 | 技術文件與 Annex IV | Model Cards + README + 架構文件 |
| Art. 12 | 自動記錄（logging） | MLflow + Prometheus + 推論日誌 |
| Art. 13 | 透明度與使用說明 | Model Card 的 Intended Use / Out-of-scope |
| Art. 14 | 人工監督 | human-in-the-loop 機制 |
| Art. 15 | 準確度、穩健性、資安 | 品質門檻（M5）+ 漂移監控（M6） |

## 4. 風險緩解措施（Mitigations）

1. **分布漂移監控**：Evidently 每日漂移報告 + Prometheus 即時告警（`monitoring/`）。
2. **品質門檻**：M5 CI/CD 在註冊前強制指標門檻，不達標不上線。
3. **人工複核**：高影響決策（停機、報廢）一律人工確認。
4. **可追溯**：模型版本、資料版本、reference 快照三者綁定，可回溯任一預測的訓練脈絡。
5. **定期再評估**：用途或資料來源變更時，重跑本風險評估。

## 5. 結論

本專案三個 AI 系統於目前用途下屬**最小至有限風險**，主要義務為**透明度、紀錄留存與人工監督**，
已透過 Model Cards、Datasheets、MLflow/DVC 追蹤與監控閉環落實。若視覺檢測延伸至**安全關鍵零件
合格性判定**，須重新評估是否升級為高風險並補齊 Art. 9–15 義務。

| 審查項目 | 狀態 |
| :--- | :--- |
| 風險分級完成 | 是 |
| 高風險義務缺口盤點 | 是（第 3.2 節） |
| 緩解措施定義 | 是（第 4 節） |
| 再評估觸發條件 | 是（用途/資料變更） |
