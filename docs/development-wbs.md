# 課程開發 WBS（Work Breakdown Structure）

> **範圍**：開發「MLOps 入門 12 小時課程」的全部交付物 —— 教材（投影片 + 講義）、capstone 程式骨架、Lab 解答、講師指引。
> **不含**：實際授課、學員端基礎設施採購。
> **工時單位**：人時（h）。估算以 1 位熟悉 MLOps 的講師/工程師為基準；標 ⚡ 者可平行。
> **相依**：以 WBS 編號表示前置工作。

---

## 1. WBS 總覽（階層分解）

```
1.0 課程開發專案
├── 1.1 專案啟動與基礎建設        (Phase 0)
├── 1.2 Capstone 程式骨架          (Phase 1)
├── 1.3 模組教材開發 M1–M6         (Phase 2)
├── 1.4 Lab 驗證與 dry-run         (Phase 3)
└── 1.5 交付、迭代與維護           (Phase 4)
```

## 8. 關鍵相依與風險

| 風險 | 影響 | 緩解 |
| :--- | :--- | :--- |
| **教材依附骨架** | Phase 2 卡在 Phase 1 | 1.2.3/1.2.4/1.2.5 平行先行，教材隨完成模組滾動開發 |
| **GPU 取得** | M4 影像 Lab 無法跑 | 1.4.3 預先備好 CPU + 量化 + 小模型後援 |
| **工具版本漂移** | MLflow/Feast/Bento 升版破壞範例 | 1.1.3 鎖版 + lockfile；維護計畫納入定期升版 |
| **2h/模組超時** | 現場講不完 | 1.4.2 彩排；把 Triton/Terraform/Grafana 標為可砍的 Demo |
| **資料授權** | 案例不能公開使用 | 1.1.2 優先選公開資料集（C-MAPSS / MVTec AD / Kaggle） |
